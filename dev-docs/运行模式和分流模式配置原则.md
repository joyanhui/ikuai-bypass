# 运行模式与分流模式配置原则

## 原则

1. **config.yml 是唯一持久化来源** — `run-mode` 和 `mode` 作为顶层字段写入 config.yml，不写入服务文件、环境变量或独立文件
2. **CLI 参数可以覆盖** — `-r` / `-m` 在运行时可以覆盖 config.yml 中的值，但不持久化
3. **服务文件不嵌入运行参数** — systemd/init.d 中的 ExecStart 不应包含 `-r` 或 `-m`，二进制启动时自动从 config.yml 读取
4. **三优先级** — CLI 参数 > config.yml > 硬编码默认值
5. **改配置 = 改 config.yml + 重启服务** — 用户修改这两个字段后必须重启服务进程才能生效

## 解析算法

```
输入: CLI 参数 (-r, -m), config.yml (run-mode, mode 字段)

处理:
1. 加载 config.yml → 得到 config.run_mode, config.module
2. 解析 CLI 参数 → 得到 args.run_mode (Option), args.module (Option)
3. 确定有效值:
   effective_run_mode = args.run_mode
       ?? (config.run_mode.not_empty ? config.run_mode : None)
       ?? "cronAft"
   effective_module = args.module
       ?? (config.module.not_empty ? config.module : None)
       ?? "ispdomain"
4. 验证 effective_module 是否合法
   (允许值: ispdomain / ipgroup / ipv6group / ii / ip / iip)
5. 后续所有逻辑使用 effective_run_mode 和 effective_module

注意:
- validate_module() 在有效值确定之后调用，而不是在原始 CLI 参数上
- config.yml 中字段缺失或为 "" 等价于"未设置"
- ?? 为 Option.or() 或 Option.or_else() 语义
```
